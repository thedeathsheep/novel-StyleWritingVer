# StyleEvent 改造阶段与文件对照

## 阶段任务速查

**阶段 1 — 布局与灵感区（不接真实检索）**
- 主布局：品牌 + 灵感区 + 编辑区 + 底部状态。
- 灵感区组件：纯文本 + 出处，背光+浮动，空态/占位文案。
- 产出：`/` 打开即该布局，灵感区可写死或 mock。

**阶段 2 — 触发逻辑打通**
- 从编辑器取当前句/词或选中文本。
- 停顿 1.5s 或选区变化触发；取消上一请求。
- 触发后传入查询文本，灵感区展示 mock 列表。

**阶段 3 — RAG 接口与契约**
- API 契约：`query` → `{ items: { text, source }[] }`。
- 服务端：embedding + 向量检索 top-K，超时 2s 返回空。
- 前端：2s 超时、取消忽略、空/错误时占位展示。

**阶段 4 — 体验与可观测（可与 2/3 并行）**
- 加载态、新条顶部插入与上滑、最大条数、延迟打点、弱化与写作无关的 UI。

## 风险与依赖

- 编辑器实例：触发逻辑必须在能拿到 `editor` 的层级（EditorRoot 内或通过 context/props 传入）。
- RAG：MVP 依赖云端 embedding + 向量库或 mock；密钥用环境变量，不写死。
- 包管理：根目录 pnpm；无 pnpm 时可在 `apps/web` 下 `npm run dev`。
